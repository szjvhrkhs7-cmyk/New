# FinPlanner — Финансовый планировщик

Progressive Web App для еженедельного планирования расходов с учётом план/факт,
аналитикой и экспортом. Устанавливается на главный экран iPhone через Safari
(«Добавить на главный экран») как полноценное автономное приложение.

## Стек

- Vanilla JavaScript (ES-модули, `type="module"`), без сборщика и без
  npm-зависимостей в рантайме — только нативные браузерные API.
- `localStorage` — локальное хранилище данных (без бэкенда).
- Service Worker (`sw.js`) + `manifest.webmanifest` — офлайн-кэш и
  устанавливаемость («Add to Home Screen») на iOS/Android.
- Инлайновые SVG-функции для графиков аналитики (без библиотек чартов).
- `window.print()` + CSS `@media print` — экспорт в PDF через нативный
  диалог печати/сохранения браузера (без PDF-библиотек).
- WebAuthn (`navigator.credentials`, платформенный аутентификатор) —
  клиентский аналог Face ID/Touch ID для блокировки приложения.
- `.ics`-файлы (RFC 5545) — практический аналог двусторонней синхронизации
  с календарём: одностороннее добавление события через системный диалог
  «Добавить в календарь» (полноценный EventKit-доступ из браузера невозможен).
- Node.js встроенный test runner (`node --test`) — модульные тесты бизнес-логики.

## Структура проекта

```

src/

  styles.css                   Дизайн-система (light/dark, акцент #5AC8FA/#0A84FF)
  core/                        Код, не привязанный к конкретной фиче
    expenseCategory.js         Enum категорий, заголовки/иконки/цвета
    expenseStatus.js           Enum статусов (запланировано/состоялось/отменено)
    expense.js                 Фабрика записи о расходе
    week.js                    Арифметика недель (понедельник — начало недели)
    currency.js                Форматирование сумм (ru-RU, RUB)
    planFact.js                Расчёт план/факт сумм и разбивки по категориям
    sort.js                    Варианты сортировки списка расходов
    modal.js                   Общий хелпер модальных окон/шторок
  state/
    store.js                   ExpenseStore/settingsStore — localStorage + pub-sub
  features/                    Код, сгруппированный по фиче
    expenses/                  Главный недельный экран (план/факт по тратам)
    calendar/                  Экспорт событий в .ics (аналог Calendar sync)
```

### Принцип модульности по фиче

Каждая фича (`expenses`, `calendar`, `analytics`, `export`, `security`,
`settings`) — самодостаточная папка со своим `render(container, ctx)` и
вспомогательными модулями. Общий код, используемый всеми фичами (модели
данных, дизайн-система, форматирование), лежит в `core`. Изменения внутри
одной фичи не должны требовать правок в другой, кроме точек интеграции через
`core` и `state/store.js`.

## Архитектурные решения

- **Без виртуального DOM**: `app.js` при каждом изменении состояния вызывает
  `render(container, ctx)` нужной фичи, который перестраивает innerHTML
  целиком. Модальные окна (`core/modal.js`) монтируются напрямую в
  `document.body`, а не в перерисовываемый контейнер, чтобы не быть
  уничтоженными посреди взаимодействия пользователя.
- **`planFact.js`** — чистые функции подсчёта план/факт сумм и разбивки по
  категориям, не зависящие от DOM/состояния, тестируются напрямую.
  `actualTotal` учитывает только завершённые траты, `plannedTotal` — все.
- **`.ics`-экспорт вместо EventKit**: `features/calendar/icsExporter.js` —
  чистая функция генерации RFC5545-календаря, `calendarDownload.js` триггерит
  скачивание/открытие в приложении «Календарь». Это сознательно одностороннее
  решение — настоящая двусторонняя синхронизация с системным календарём
  недоступна из браузера/PWA без бэкенда и нативных разрешений.
- **WebAuthn-блокировка вместо LocalAuthentication**: `features/security/webauthnLock.js`
  использует `navigator.credentials.create/get` с платформенным
  аутентификатором как клиентский гейт без relying-party сервера — ближайший
  достижимый аналог Face ID/Touch ID на вебе. Это не полноценная серверная
  WebAuthn-аутентификация, а локальная проверка присутствия пользователя
  через биометрию устройства.
- **PDF через `window.print()`**: `features/export/printReport.js` заполняет
  скрытый `#print-area`, а CSS-правила `@media print` в `styles.css` прячут
  остальной интерфейс и показывают только отчёт — пользователь сохраняет PDF
  через нативный диалог печати/«Поделиться» браузера, без PDF-библиотек.
- **Service worker (`sw.js`)** кэширует весь app shell (HTML, CSS, все JS-модули,
  иконки) при установке и обслуживает запросы по стратегии
  network-first-with-cache-fallback — приложение остаётся рабочим офлайн.

## Конвенции именования и стиля

- UI-строки — на русском языке (это продукт для русскоязычного рынка).
  Имена функций, файлов, переменных — на английском.
- Имя файла = имя главного экспорта/модуля в нём.
- Комментарии пишутся, только когда без них неочевидна причина решения
  (нестандартный обход API, хрупкий инвариант). Не описывать в комментариях,
  что делает код — это видно из именования.
- Группировка: `xxxView.js` — рендер экрана/подэкрана, `xxxForm.js`/`xxxSheet.js` —
  модальные формы, чистая логика — функции расчёта без суффикса класса (этот
  проект не использует классы для бизнес-логики, только для `ExpenseStore`).
- Состояние UI (вкладка, выбранная неделя, сортировка, период аналитики)
  держится в одном объекте `uiState` в `app.js` и передаётся фичам через
  `ctx.uiState`/`ctx.setUiState` — не дублируется в фичах.

## Тестирование

`tests/` использует встроенный test runner Node.js (`node:test`,
`node:assert/strict`) и покрывает только чистую логику, не зависящую от DOM:

- `week.test.js` — арифметика недель (`core/week.js`).
- `planFact.test.js` — расчёт план/факт и разбивки по категориям.
- `weeklyTrend.test.js` — разбивка трат по неделям для графика тренда.
- `csvExporter.test.js` — формат CSV, сортировка, экранирование полей.
- `icsExporter.test.js` — формат `.ics`, экранирование текста по RFC5545.

Новый код с нетривиальной логикой (расчёты, парсинг, форматирование) должен
сопровождаться тестами в этом стиле: чистая функция без DOM/localStorage,
тестируется изолированно.

## Сборка и тесты

Без сборщика и без зависимостей — приложение раздаётся как статические файлы
(`index.html` + `src/` + `manifest.webmanifest` + `sw.js` + `icons/`) любым
статическим веб-сервером или просто открытием `index.html`.

```bash
npm test          # node --test tests/
```
